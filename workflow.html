  Expected: 客戶名稱自動完成顯示建議

  Expected: 客戶資訊自動填充

  Expected: 金額正確顯示

Step 7: 點擊「提交」按鈕
  Expected: 訂單成功建立，返回訂單列表

📸 視覺分析

💾 知識積累

將已知的頁面和元素存儲在 定位器快取中以加速未來的測試

第3階段：代碼生成

⚙️ 自動化代碼合成

探索完成後，AiScripto使用收集的UI資訊生成 符合行業最佳實踐的Playwright TypeScript代碼。

📦 生成輸出

📄 tests/TC-*.spec.ts

完整的Playwright測試規範

📚 src/pages/*.ts

頁面物件模型（POM）

🔗 fixtures/*.ts

共享fixtures（登入、設置等）

📊 *.data.json

參數化測試數據

📋 report.json

執行報告和分析

生成代碼的特點：

  • 型別安全
    完整的TypeScript型別定義
  • 可讀性高
    清晰的變數名和邏輯流程
  • 可維護性
    遵循Playwright官方最佳實踐
  • 錯誤處理
    完善的異常捕獲和日誌
  • 性能優化
    智能等待和超時配置
  • 可測試性
    可以直接運行和修改

生成的測試代碼示例

import { test, expect } from '@playwright/test';
import { OrderPage } from '../pages/OrderPage';

test.describe('訂單管理測試組', () => {
  let page;
  let orderPage: OrderPage;
  
  test.beforeEach(async ({ page: p }) => {
    page = p;
    orderPage = new OrderPage(page);
    
    // 登入應用
    await page.goto('https://app.example.com/login');
    await page.getByRole('textbox', { name: /用戶名/ }).fill('testuser');
    await page.getByRole('textbox', { name: /密碼/ }).fill('password');
    await page.getByRole('button', { name: /登入/ }).click();
  });
  
  test('TC-001: 建立新訂單', async () => {
    // Step 1: 導航至訂單管理頁面
    await page.goto('https://app.example.com/orders');
    await expect(page.getByRole('table')).toBeVisible();
    
    // Step 2: 點擊建立新訂單按鈕
    await orderPage.clickCreateButton();
    await expect(page.getByRole('dialog')).toBeVisible();
    
    // Step 3-7: 填充表單並提交
    await orderPage.fillOrderNumber('ORD-2024-001');
    await orderPage.fillCustomerName('ABC Corporation');
    // ... 更多步驟
    
    // 驗證結果
    await expect(
      page.getByText(/訂單已成功建立/)
    ).toBeVisible();
  });
});

第5階段：參數化（可選）

📊 數據驅動測試轉換

完成基本測試後，可以選擇將硬編碼的測試數據提取到JSON文件， 使同一測試能用多組數據執行。

提取過程：

  • 1. 識別數據
    AI掃描代碼找出所有硬編碼值
  • 2. 分類數據
    按類型分組（訂單、客戶、金額等）
  • 3. 生成數據文件
    創建JSON結構化數據
  • 4. 參數化代碼
    用變數替換硬編碼值
  • 5. 驗證執行
    用所有數據組合執行測試

轉換示例

Before (硬編碼)：

await orderPage.fillOrderNumber('ORD-001');
await orderPage.fillAmount(5000);
await orderPage.fillCustomer('ABC Corp');

After (參數化)：

await orderPage.fillOrderNumber(testData.orderId);
await orderPage.fillAmount(testData.amount);
await orderPage.fillCustomer(testData.customer);

測試數據文件結構

// tests/data/order-creation.data.json
{
  "testCases": [
    {
      "id": "ORD-001",
      "orderId": "ORD-2024-001",
      "customer": "ABC Corporation",
      "amount": 5000,
      "expectedStatus": "成功建立"
    },
    {
      "id": "ORD-002",
      "orderId": "ORD-2024-002",
      "customer": "XYZ Limited",
      "amount": 15000,
      "expectedStatus": "成功建立"
    },
    {
      "id": "ORD-003",
      "orderId": "ORD-2024-003",
      "customer": "QRS Industries",
      "amount": 8500,
      "expectedStatus": "成功建立"
    }
  ]
}

結果： 1個測試 × 3個數據組合 = 3次完整的測試執行， 覆蓋不同的業務場景

📋 工作流程命令參考

命令 功能 何時使用 輸出
/import 導入測試用例 添加新的測試用例到系統 imported-cases.json
/validate 驗證測試格式 檢查測試用例語法和完整性 validation-report.json
/explore 探索UI並收集元素 創建新測試或更新過時的定位器 locators.json, screenshots/
/generate 生成Playwright代碼 從測試用例創建可執行測試 tests/TC-*.spec.ts, pages/
/execute 運行測試 驗證生成的測試是否通過 test-results.json, traces/
/heal 自動修復失敗測試 修復測試失敗而不重新編寫 healed-tests/, healing-report.json
/parameterize 提取數據並參數化 將硬編碼數據轉為數據驅動 *.data.json, parameterized-tests/
/verify 最終驗證 使用所有數據組合重新執行 final-verification-report.json
