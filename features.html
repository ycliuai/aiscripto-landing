自動化頁面物件模型生成

📚 業界最佳實踐

AiScripto自動生成遵循Playwright官方最佳實踐的頁面物件模型（POM）。 無需手動編寫和維護元素選擇器。

🎯 可訪問的定位器

優先使用ARIA角色和標籤，而非脆弱的CSS選擇器

♻️ 可重用組件

生成的POM可在多個測試中重複使用

🔍 定位器快取

建立知識庫，加速後續頁面的測試生成

📝 完整文檔

自動生成的代碼包含注釋和型別定義

範例：自動生成的頁面物件

// 自動生成的頁面物件模型
export class OrderPage {
  readonly page: Page;
  
  // 訂單編號欄位
  readonly orderNumberInput = this.page.getByRole(
    'textbox', { name: /訂單編號/ }
  );
  
  // 建立按鈕
  readonly createButton = this.page.getByRole(
    'button', { name: /建立/ }
  );
  
  // 訂單表格
  readonly orderTable = this.page.getByRole(
    'table', { name: /訂單清單/ }
  );
  
  async fillOrderNumber(number: string) {
    await this.orderNumberInput.fill(number);
  }
  
  async clickCreate() {
    await this.createButton.click();
  }
}

數據驅動測試

📊 外部化測試數據

AiScripto可以自動識別硬編碼的測試數據，並將其提取到JSON文件中。 這使得同一測試可以用不同的數據組合執行。

功能優勢：

  • 多數據組合
    單一測試用例執行多組數據
  • 易於管理
    集中管理所有測試數據
  • 批量執行
    並行執行相同的測試流程
  • 易於更新
    無需修改代碼即可更新數據

[
  {

  {

    "amount": 12000
  }
]
